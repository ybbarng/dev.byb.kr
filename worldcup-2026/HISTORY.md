# 개발 여정 — 데이터로 본 2026 월드컵

한 페이지짜리 월드컵 예측 사이트가 **한 달간 매일 발행되며 자란 기록**입니다. 대회가 시작되기 전부터 끝난 뒤 회고까지, 무엇을 만들었고 어디서 넘어졌는지를 시간순으로 남깁니다. (원본 작업 저장소 `~/worldcup`은 대회 종료 후 폐기했고, 이 문서가 그 맥락을 대신합니다.)

---

## 0. 시작 — git 이전 (6월 24~26일)

처음엔 git 없이 HTML 한 장으로 출발했습니다. **"대한민국이 32강에 진출할 확률은?"** 이라는 질문에서 시작해, FIFA 랭킹을 Elo 점수로 바꾸고 조별리그부터 토너먼트까지 **50만 회 몬테카를로 시뮬레이션**을 돌려 확률을 냈습니다. 6/24·25·26 세 번의 예측이 이 시기에 나왔고(페이지 `VERSIONS` 배열의 가장 오래된 항목들), 경우의 수·응원 보드·움직이는 미니 경기장 같은 요소도 이때 붙었습니다.

## 1. git 관리 시작 (6월 27일)

여기서부터 버전 관리를 시작했습니다. 첫 커밋이 이미 1.8만 줄이었을 만큼, git 이전에 상당히 만들어진 상태였죠. 이 날은 주로 **다듬기**였습니다:

- 모바일 경기장 물리·비율 통일, 스크롤 성능(관객석 재그리기 생략, `backdrop-filter` 제거, `transform:translate3d`로 reflow 제거)
- GOAL 애니메이션 GPU 합성, 점수판 위치
- 회사/개인 산출물을 나누는 `build.py`(당시엔 사내 배포도 했음)

## 2. 한국의 탈락, 그리고 방향 전환 (6월 28~29일)

대한민국이 32강에서 탈락하면서 페이지의 의미가 바뀌었습니다. "한국 진출 확률"이 아니라 **"남은 토너먼트 전체를 예측하고 되돌아보기"**로 방향을 틀었죠.

- 한국 페이지 + 라운드 탭(32강~결승) 구조로 개편, 탭 상태를 **URL 해시**로 유지
- **모델 캘리브레이션**: 처음엔 강팀이 약팀을 지나치게 압도했습니다(아르헨티나 우승 57%). 골 기대값에 `tanh` 기반 비선형 상한을 걸어 실제 결과에 맞게 보정(→ 42%)
- **조건부 우승 확률** 추가: "우승 확률"과 "결승에 올랐을 때의 승률(대진운을 걷어낸 실력)"을 구분해 보여줌
- 이후로는 **회사(buzz-html) 배포를 접고 개인 배포만** 하기로 결정

## 3. 매일의 발행 — 32강부터 결승까지 (6월 30일 ~ 7월 20일)

대회가 진행되는 동안 **경기 결과가 나올 때마다 그날의 버전을 발행**했습니다. 각 발행은 `wc_state_<날짜>.json`(그날까지의 확정 결과)을 입력으로 시뮬레이션을 다시 돌려, 페이지의 `VERSIONS` 배열 맨 앞에 새 버전을 끼워 넣는 방식이었습니다.

- 32강(6/30~7/4) → 16강(7/5~7/8) → 8강(7/10~12) → 준결승(7/15~16) → 3·4위전·결승(7/19~20)
- 라운드가 끝날 때마다 **기본 진입 탭**을 다음 라운드로 옮김(32강→16강→8강→4강→결승)
- 각 경기 카드에 **예상 vs 실제 스코어**, 예상 적중/빗나감을 표시. 승부차기 스코어까지
- "볼 만한 경기·일방적인 경기·골 잔치" 같은 흥미 요소에도 실제 결과 대조를 붙임
- **확정된 경기는 예측의 출발점으로 고정**하도록 시뮬을 고침 — 이미 이긴 팀을 계속 "이길 수도 있다"고 예측하던 문제를 바로잡음

🏆 결과: **스페인 우승** · 🥈 아르헨티나 · 🥉 잉글랜드 · 4위 프랑스.

## 4. 코드가 자라며 겪은 것들

한 파일이 커지면서 부딪힌 것들, 그리고 배운 것들:

- **Group I 오기입 사고**: 조별 최종전(France vs Norway)이 아직 안 끝났는데 "공식 종료"라 적고 Norway를 1위로 넣었다가, 실제 대진이 어긋남. 이후 **"결과를 넣기 전 웹으로 재확인"하는 게이트**를 절차로 못박음.
- **매치 85 오보**: 한 뉴스 기사만 믿고 "알제리 승"을 넣었는데 실제는 "스위스 2-0". 이후 **FIFA 공식 + 복수 출처 교차확인**을 원칙으로. (게이트 덕에 발행 전에 잡음)
- **`.miss` 클래스 충돌**: 예측 격자의 셀에 `miss` 클래스를 썼는데, 전역 골 애니메이션 클래스(`position:fixed`)와 충돌해 셀이 화면 중앙으로 날아감. 고유 접두사(`sm-`)로 해결. (같은 함정을 두 번 밟고서야 배움)
- **데드코드 정리**: 기능을 갈아끼우며 남긴 죽은 함수들(`interestSec`·`simHTML` 등)을 정적 스캔+수동 확인으로 제거. 이 과정에서 "통짜 HTML을 프레임워크로 바꿀까"를 고민했지만, **폐기 예정 프로젝트라 통짜 유지**로 결정.
- **빌드 단순화**: 회사 분기·사내 로그인 코드·발행본 백업을 차례로 걷어내, 결국 **"정본 HTML 한 장이 곧 배포본"**이 되게 함.

## 5. 대회의 끝, 그리고 회고 (7월 20일)

결승까지 끝난 뒤, 단순히 결과만 보여주는 대신 **"우리 예측이 얼마나 맞았나"를 회고하는 총정리 탭**을 만들었습니다.

- **예측 적중 지도**: 가로=발행 날짜, 세로=경기. 각 발행에서의 예측이 최종 결과와 맞았는지를 녹/빨 격자로. 확정된 칸은 비워, 오른쪽·아래로 갈수록 삼각형이 됨.
- **인사이트**: 라운드별 적중률(8강 75%·4강 83%로 정점 → 결승 0%), "아르헨티나의 역설"(결승行은 완벽히 맞혔으나 결승 패배는 20번 내내 못 봄), 끝내 못 맞힌 이변들(노르웨이·스위스·스페인 등)
- **왜 못 맞혔나**: 방법론(FIFA 랭킹→Elo→골 기대값→50만 회 시뮬)을 도식으로 설명하고, **Elo를 대회 내내 고정한 경직성**을 데이터로 드러냄(브라질전 11번·결승 20번 내내 같은 예측). 개선안(Elo 동적 갱신)과 "단판 토너먼트는 이변이 상수"라는 결론까지.
- 전체 예측 적중률은 **약 62%**. 동전 던지기보단 낫지만, 정작 우승을 못 맞힌 건 이 대회가 그만큼 드라마틱했다는 뜻.

같은 날, 대회 종료로 **실시간 응원 보드(구글 스프레드시트/Apps Script) 연동을 제거**하고 그동안의 응원을 하드코딩으로 남겼습니다.

## 6. 아카이브 — dev.byb.kr로

대회가 끝나 이 페이지를 개인 허브 **[dev.byb.kr](https://dev.byb.kr)**로 옮겼습니다. 발행 파이프라인·시뮬레이션 코드는 가치를 다해 폐기하고, **결과물(예측 데이터가 박힌 HTML 한 장)과 이 여정 기록만** 남깁니다. 옛 주소 `byb.kr/korea-wc2026.html`은 `dev.byb.kr/worldcup-2026/`으로 301 리다이렉트됩니다.
